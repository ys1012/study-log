この4つに分けて解説していきます

## 【GitHubの準備】

・GitHub上にリポジトリを作成
『送り先』を作っておきます、ログインして
「Create a new repository」からリポジトリを作成し
「Repository name」を設定してください
公開設定は「Choose visibility」からやっておきましょう
呪文のようなものが書いてあるページが現れますが、これを参考にVSCodeのターミナルに設定していきます

・メールアドレスの隠蔽の設定　※推奨※
これをしないと、公開したコードの履歴からメールアドレスが見えます
GitHubの Settings > Emails から Keep my email addresses private をオンにしておきましょう
※発行された`ID+username@users.noreply.github.com`というダミーのアドレスを、後ほどの設定で使用します

## 【フォルダの準備】
PCに共有するフォルダ（PC上のリポジトリ）を作成しておきましょう
この中のものをGitHub上のリポジトリへアップロードすることができます。

※中身が何もない状態だとこの後の手順を踏んでもエラーとなります。

おすすめとしては「README.md」というファイルを作成して、なにか一言かいておきましょう
この一言がGitHub上リポジトリのREADME欄にのります。

## 【VS Codeの準備】

左上のファイルからフォルダを開き共有するフォルダを選択しておきます。

※私はこれをしなかったので、意図しないVS Codeの設定ファイルがGitHub上のリポジトリへ入ってしまいました。

それが終わったら表示タブからターミナルを開いておきましょう


## 【Gitの準備】
ここからターミナルへコマンドを入力して設定をおこないます。

・私が誰であるかの登録
※ここで発行されたダミーのアドレスを入力します
```
git config --global user.name "YourName"
git config --global user.email "ID+username@users.noreply.github.com"
```


・PC上のリポジトリを初期化
```
git init
```

・ブランチ名の変更
※標準だとmasterになっているので、GitHubの標準である「main」に強制変更します。
```
git branch -M main
```

・GitHub上のリポジトリとの紐付け
※自分のGitHubのアドレスに書き換えてください､それにOriginという略称をつけます。
```
git remote add origin https://github.com/ユーザー名/リポジトリ名.git
```

・次に荷造りの段階（ステージング）
※「.」はフォルダ内全部という意味で、全部荷物にいれますという意味
```
git add .
```

・荷造りを終了し、荷札（メモ）を貼り付けておきます（コミット）
※メモ部分はなんでもOKですが、何をしたかわかるようなものを推奨します。
```
git commit -m "メモ"
```
・GitHub上のリポジトリへ発送（プッシュ）
※「-u」はmainブランチの標準の宛先はoriginにしておくよという意味
```
git push -u origin main
```
・最後にGitHubを確認
更新されていたら成功です。
もし失敗した時は原因を特定して修正することが必要ですが、新しく組み立て直すことも考えてみてください。


## 【今後更新するには】
初期設定は済んでいるので、今後の更新では
「荷造り」→「荷札貼付」→「発送」
ステージング→コミット→プッシュの3ステップを実行します。
※メモはなんでもOKですが、何をしたかわかるようなものを推奨です。
```
git add .
git commit -m "メモ"
git push
```
ちなみにVSCodeの機能を利用してGUI操作で更新も可能です。
でもコマンド操作ってなんかロマンがありますよね
